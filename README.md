## 一、先分清你要的是 MVP 还是 SaaS

这俩词在外包群里经常被混着喊，但报价能差三倍。先对齐概念，不然你拿着 MVP 的预算去问 SaaS，要么被劝退，要么被忽悠加价。

| | **MVP（最小可行产品）** | **SaaS（可持续运营的产品）** |
|---|---|---|
| 目的 | 验证「这事有没有人要」 | 已验证，要稳定收钱、长期迭代 |
| 用户量级 | 几十到几百个种子用户 | 成百上千付费账户 |
| 核心功能 | 1 条主流程能跑通就行 | 多角色权限、计费、对账、后台 |
| 该不该上来就做得很全 | **不该**，越省越好 | 要，但仍要分期 |
| 典型周期 | 1–2.5 个月 | 2–4 个月起 |
| 典型预算 | 1–2 万 | 2–3 万 |

一句话：**MVP 是用来「被证伪」的，所以越便宜越快越好；SaaS 是要「被运营」的，贵在地基。** 90% 的早期创始人其实该做的是 MVP，却总想一步到位做 SaaS，这是最常见的烧钱方式。

## 二、2026 外包报价锚点（带价格区间）

下面是 2026 年国内一线城市、用现代技术栈（Next.js / React 这类）交付的**大致行情**。注意：这是「锚点」不是「报价单」，真实项目按范围（功能数、角色数、第三方集成、设计复杂度）浮动，差一倍都正常。

| 产品类型 | 价格区间 | 典型周期 | 说明 |
|---|---|---|---|
| 简单 MVP | **1–2 万** | 1–2.5 个月 | 单一主流程 + 基础后台，验证想法用 |
| 标准 SaaS 产品 | **2–3 万** | 2–4 个月 | 多角色 + 计费 + 后台，能开始收钱 |
| 含 AI 集成 | **3–8 万** | 看范围 | 接大模型 / RAG / Agent，集成与调优是大头 |

**怎么用这张表：**
- 有人**几千块**就敢接一个"完整 SaaS"——别光看便宜。这种多半是套模板、或被层层转包给学生练手，最后拿个半成品（坑见第五节）。一两千、两三千这种价，基本可以直接划掉。
- 反过来，有人开口**十几万、几十万做个早期 MVP**——要么把你当冤大头，要么是企业级团队的起做门槛（见第四节），跟你"先花小钱验证想法"的阶段根本不匹配。
- 真正合理的报价，应该**附带范围拆解**：哪几个页面、哪几个角色、接哪几个第三方、不含什么。报价不带范围的，等于没报价。

> 提醒：以上是「开发费」。别忘了还有**持续成本**——服务器 / 数据库 / 域名 / 短信 / 大模型 API 调用，这些是上线后每月要花的，很多人做预算时漏掉，结果产品做好了养不起。

## 三、为什么 2025–26 价格在跌：AI 把行情打下来了

如果你 2022 年问过价、现在重新问，会发现便宜了不少。这不是错觉。

Cursor、GitHub Copilot 这类 AI 编程工具，让一个熟练工程师的产出效率明显提高，**写样板代码、搭脚手架、写测试**这些过去最耗工时的活儿被压缩了。结果就是**外包定价整体下行约 20%–50%**（看活儿的性质，越标准化的降得越多）。

这对甲方是好事，但有两个副作用你得知道：

1. **「便宜」不等于「省心」。** AI 能加速写代码，但**需求对齐、架构决策、上线运维、出 bug 时的判断**这些没法外包给 AI。报价里被砍掉的是码字工时，不是这部分脑力——谁把这部分也砍没了，谁就是在埋雷。
2. **警惕「纯 AI 糊出来」的交付。** 现在有人用 AI 一把梭生成一坨能跑但没人能维护的代码，验收时看着没问题，三个月后想加功能发现没人敢动。所以**代码可维护性、有没有人能接手**，比当下「跑不跑得起来」更重要。

我自己的做法是把 AI 当提效工具用，省下来的时间花在陪你把需求想清楚上——这也是为什么我敢把价钱压在那个区间还能保证质量。

## 四、玩家横评：谁适合你，谁别碰

市面上接 SaaS / MVP 的玩家大致分五类，各有各的活法。我尽量客观，每类都写清楚**适合谁**和**坑在哪**。

### 1. 企业级老牌外包（盛安德、信必优 等）

- **优点**：流程规范、交付稳、有大客户背书，适合预算充足的中大型企业做严肃系统。出问题有正规主体兜底。
- **坑**：**贵，而且门槛高**——最小起做基本在 **30 万 +**，再小的单子人家不接，或者接了也不上心。流程重、响应慢、不接地气，一个早期创始人拿着几万块想做 MVP 去找他们，多半被劝退。
- **适合**：已经有钱有规模、要做核心业务系统的公司。**不适合**验证想法阶段的小团队。

### 2. 撮合平台（程序员客栈 等）

- **优点**：能快速找到个人开发者，价格相对透明，小活儿、临时活儿方便。
- **坑**：本质是**中介撮合**，平台要抽成——综合算下来**甲方侧成本约 23%** 是付给平台的，不是给干活的人。更要命的是**团队没有延续性**：这次这个人、下次另一个人，项目交接全靠你自己盯。出了纠纷平台只能调解，不真正对结果负责。
- **适合**：模块化的小任务、明确边界的外包件。**不适合**需要长期迭代、一个团队从头跟到尾的产品。

### 3. 低价众包（猪八戒 这类）

- **优点**：便宜，是真便宜。预算极其有限、要求极其简单时能用。
- **坑**：**质量是赌博**。同样标价，可能遇到靠谱的，也可能遇到拿你的钱练手 / 直接转包出去的。验收标准全靠你自己懂行，不懂就只能认。
- **适合**：一次性、不重要、坏了也无所谓的边角料需求。**做 SaaS / 要长期用的产品，别在这赌。**

### 4. 新兴 AI 工作室（Dify / RAGFlow 生态的小团队）

- **优点**：踩在最新的 AI 编排工具上，搭一个「能聊天 / 能检索」的 demo 很快，对想做 AI 产品的人有吸引力。
- **坑**：**太新，口碑难独立验证**。很多是这一两年冒出来的小作坊，没有可查的长期案例，团队稳定性和交付能力存疑。AI 这层迭代又快，今天的技术选型半年后可能就过时。
- **适合**：自己懂技术、能判断对方水平、想快速试 AI 方向的人。**不适合**完全外行、把宝全押上去的。

### 5. 离岸团队（越南 / 东欧的 Next.js 团队）

- **优点**：技术不错，英文沟通顺，按小时计费透明。Saigon Technology、Golden Owl、Pagepro 这类是其中口碑较好的，单价大约 **$25–140/hr**（越南端偏低、东欧端偏高）。适合有英文能力、习惯远程协作的团队。
- **坑**：**时差 + 语言 + 文化**三道墙。需求稍微模糊一点，跨时区来回对一遍就是一天。中文语境下的业务细节（微信生态、国内支付、合规）他们往往不熟。
- **适合**：英文 OK、需求能写清楚、做面向海外产品的团队。**不适合**离不开中文沟通、需求还在变的早期项目。

### 那 1–10 万这个区间谁来接？

你会发现：上面五类里，**「1–10 万预算 + 要可验证地落地 + 要现代栈（Next.js）+ 要全程能直接对话工程师」**这个组合，几乎是真空的——大厂看不上、平台没延续、低价靠赌、AI 作坊没口碑、离岸有沟通墙。

这正是我们 [模酷 ModalCube](https://modalcube.com) 在做的事，顺手说一下我们的打法（你拿去当**验收标尺**，去衡量任何一家外包都行）：

- **透明定价**：早期 MVP 大概 **1–2 万**、一般产品 **2–3 万**、含 AI 集成的按范围报，报价都附范围拆解，不玩「先低价进来再加项」。具体多少，把需求说清了**加微信 gav1nq** 聊，先报清范围再动手。
- **代码 100% 归你**，托管在**你自己的 Git** 仓库里——随时能换人，不锁在我这。
- **现代栈 Next.js**，手机端 PageSpeed 跑 90+，不是糊个能跑就完事。
- **不锁平台**，上线后**30 天免费维护**，需要再上可选 SLA。
- **企业法人主体担责**，不是个人接私活跑路；**固定团队直接对接核心工程师**，不转包、不甩给客服。

不是说别人不好，是不同区间不同活法。你按自己的预算和需求对号入座，能省很多冤枉钱。

## 五、甲方最容易踩的 5 个坑（合同雷区）

这一节最值钱。下面每个坑我都见过真实案例翻车，**全部在合同阶段就能堵住**。

### 坑 1：代码 / 知识产权（IP）默认不归你 ⚠️

这是最隐蔽、后果最严重的一个。很多人以为「我付了钱，东西就是我的」——**法律上不是。**

根据《计算机软件保护条例》第十一条：**接受委托开发的软件，著作权归属由双方书面合同约定；没有书面合同、或合同没写清楚的，著作权归「受托人」（也就是开发方）。**

翻译成人话：**你不在合同里白纸黑字写明「代码及著作权全部归甲方」，那默认归开发者。** 后果是——你想换团队，对方可以不给源码；你想二次开发，可能构成侵权。

✅ **怎么堵**：合同里明确「**项目全部源代码、文档及著作权自交付/付清款项之日起归甲方所有**」，并约定交付时把代码推到**你自己的 Git 仓库**。

### 坑 2：验收标准模糊

「做一个用户管理系统」——这种描述等于没说。多模糊？多到对方交个能登录的页面就敢说「做完了」，你想挑毛病都没依据。

✅ **怎么堵**：把验收标准写成**可勾选的清单**——「支持手机号注册登录 / 支持微信支付 / 后台可导出 Excel / 首页手机端打开 < 2 秒……」每条都能用「是 / 否」回答。模糊形容词（好用、稳定、美观）一律换成可测量的指标。

### 坑 3：层层转包

你谈的是 A 团队，签完字活儿被转给 B，B 又外包给在校学生。每转一层，钱被抽一道、质量掉一截、沟通断一环。出了问题，A 说找 B，B 说找学生，学生已经毕业失联。

✅ **怎么堵**：合同里写「**未经甲方书面同意，乙方不得将项目整体或部分转包 / 分包**」。开工前问清楚：**到底是谁写代码？能不能直接和这个工程师对话？** 答不上来、或者只让你跟销售/客服聊的，警惕。

### 坑 4：GPL 开源协议污染

为了赶工省钱，有的开发者直接抄一个 **GPL 协议**的开源项目改改交给你。GPL 有「传染性」：**用了 GPL 代码，你的整个产品可能被要求一并开源。** 对一个想靠 SaaS 收费、靠代码建壁垒的产品来说，这是灾难——竞争对手理论上能合法拿走你全部源码。

✅ **怎么堵**：合同里要求「**交付物不得包含 GPL/AGPL 等具传染性协议的代码；使用的第三方开源组件须提供清单及其许可证（MIT/Apache/BSD 等宽松协议为宜）**」。让对方给一份**依赖清单**，这事不亏心的团队几分钟就能给你。

### 坑 5：没有里程碑付款，要么全预付要么尾款拖死

一次性全款预付 = 你失去全部主动权；纯靠口头「做完再说」= 对方失去全部安全感，活儿质量随缘。

✅ **怎么堵**：按**里程碑分期**，每期对应可验收的交付物。常见结构是 **30% 启动 / 40% 中期交付 / 30% 验收尾款**，每一笔都对着一个能验收的节点。这样双方都有约束，谁掉链子谁担。

## 六、签字前自查清单（可打印）

把这张表打印出来，对着你手上的合同 / 报价单逐条勾。**有一项打叉，就别急着签字。**

```
【范围与报价】
[ ] 报价附带功能范围拆解（页面/角色/集成/不含项写清楚）
[ ] 明确区分「开发费」和「上线后的持续成本」（服务器/API/短信等）
[ ] 按里程碑分期付款，每期对应可验收节点

【知识产权】★最重要
[ ] 合同写明：全部源代码 + 著作权归甲方
[ ] 约定交付时代码推到「甲方自己的 Git 仓库」
[ ] 第三方开源组件提供清单，无 GPL/AGPL 传染性协议

【交付与质量】
[ ] 验收标准是可勾选清单，不是模糊形容词
[ ] 明确技术栈（别接一个没人会维护的冷门栈）
[ ] 约定上线后维护期（多久、含什么、之后怎么收费）

【团队与责任】
[ ] 禁止未经同意转包/分包，写进合同
[ ] 能直接对话真正写代码的工程师，不是只对销售
[ ] 乙方是企业法人主体（出事能找到人担责）

【沟通】
[ ] 约定固定沟通节奏（周会/进度同步）和对接人
[ ] 需求变更怎么计费，提前说清楚（防需求蠕变）
```

## 七、决策树：3 分钟选对外包类型

```
你现在处于哪个阶段？
│
├─ 想验证「这事有没有人要」，还没收过钱
│   └─ 做 MVP（预算 1–2 万 / 1–2.5 月）
│       └─ 越省越好，只做 1 条主流程，别上来追求大而全
│
├─ 已验证有人愿意付费，要正经收钱、长期迭代
│   └─ 做标准 SaaS 产品（预算 2–3 万 / 2–4 月）
│       └─ 重点投地基：权限、计费、对账、可维护性
│
└─ 核心要接大模型 / RAG / Agent
    └─ 含 AI 集成（3–8 万，看范围）
        └─ 警惕「纯 AI 糊出来没人能维护」，盯交付质量

预算与团队怎么匹配？
│
├─ 预算充足(30万+) + 大型严肃系统 → 企业级老牌（稳但贵）
├─ 小任务/临时件 + 边界清晰      → 撮合平台（注意抽成≈23%、无延续）
├─ 极限预算 + 不重要的需求        → 低价众包（质量靠赌，慎用）
├─ 自己懂技术 + 想试 AI 方向      → 新兴 AI 工作室（口碑难验证）
├─ 英文OK + 做海外产品           → 离岸团队（$25–140/hr，有沟通墙）
└─ 1–10万 + 要可验证落地+现代栈   → 找一个能直接对话工程师、
                                    代码归你、不锁平台的团队
```
